## 2. Distinctive frameworks in the most relevant skills

**product-marketing (positioning hub)** — 12-section context doc: product overview; target audience + JTBD; B2B persona triad (User / Champion / Decision Maker with per-role concerns); problems & why alternatives fail; competitive landscape (direct/secondary/indirect); differentiation; top-3 objections + anti-personas; **Four Forces switching model (Push, Pull, Habit, Anxiety)**; verbatim customer-language glossary; brand voice; proof points; goals/conversion action. Workflow: auto-draft from codebase/site artifacts OR conversational walk-through, validating section by section. Key heuristic: capture verbatim customer language over polished descriptions.

**copywriting** — Headline templates: "{Achieve outcome} without {pain point}", "The {category} for {audience}", "Never {unpleasant event} again", pain-point question. CTA formula: [Action Verb] + [What They Get] + [Qualifier] ("See Pricing for My Team"). Five style rules: simple>complex, specific>vague, active>passive, confident>qualified, show>tell. Benefits-as-transformation rule ("Cut reporting from 4 hours to 15 minutes"). "If clear vs creative, choose clear." 7-section page skeleton: above-fold → social proof → problem → solution/benefits → how-it-works → objection handling → final CTA. Match headline to traffic source; mirror language from reviews/support tickets.

**cro** — 7-point audit in priority order: (1) value-prop clarity (flag feature-speak, vague positioning, jargon), (2) headline effectiveness (outcome + specificity/numbers), (3) CTA strategy (above-fold placement, primary/secondary hierarchy, "Submit" vs "Start Free Trial"), (4) visual hierarchy/scannability, (5) trust signals (attributed testimonials with photos, metrics, badges — placed near CTAs), (6) objection handling (price, implementation anxiety, "will it work for me"), (7) friction (form fields, mobile, load time). Page-type-specific rules: homepage = dual path for buyers vs researchers; landing page = traffic-source match + single CTA + stripped nav; pricing = recommended-plan highlight; blog = inline contextual CTAs. Output triaged as Quick Wins / High-Impact Changes / Test Ideas, with rewritten copy alternatives.

**emails** — Principles: "One Email, One Job" (single CTA), "Value Before Ask." Concrete sequence templates with counts + timing: **Welcome (5-7 emails / 12-14 days)**: welcome → quick win (d1-2) → story/positioning (d3-4) → social proof (d5-6) → objections (d7-8) → core feature (d9-11) → conversion ask (d12-14). **Lead nurture (6-8 / 2-3 wks)**: delivery → topic expansion → problem deep-dive → solution framework → case study → differentiation → objection resolution → direct offer. Re-engagement (3-4 / 2 wks, triggered at 30-60d inactivity); onboarding (5-7 / 14d). Subject lines: 40-60 chars, preview text 90-140; clarity>cleverness; question/how-to/numbered/story-tease patterns. Body: hook–context–value–CTA–signoff; length bands (transactional 50-125w, educational 150-300w, story 300-500w). Architecture: exit conditions, segmentation, B2B-avoids-weekends timing.

**customer-research (ICP/segmentation)** — Two modes: (1) synthesize existing assets (transcripts, tickets, win/loss, NPS), (2) "digital watering hole" mining (Reddit, G2, HN, LinkedIn). JTBD split: functional/emotional/social jobs. Extraction categories: pain points (weight unprompted, emotional mentions), trigger events, desired outcomes verbatim, alternatives considered, vocabulary. **Confidence labeling: High = 3+ independent sources + unprompted + cross-segment consistent; Medium = 2 sources or segment-specific; Low = single source.** Persona minimum: 5-10 data points per segment; persona schema = triggers, pains, outcomes, objections, alternatives, vocabulary, channels.

**marketing-plan (GTM)** — The deepest skill. 13-section plan organized around **AARRR funnel stages instead of channel lists** — every tactic must name which stage it moves. 17-section current-state rubric scored 0-5. **Funding-stage capability unlocks** (bootstrapped $0-2K/mo organic-only → seed $5-15K paid testing → Series A $50-150K → B+ $150K+). S-curve growth phases ($0-10K / $10K-100K / $100K-1M ARR) gate channel choices. Budget formulas: 5-40% of ARR, or reverse-engineered `[(New ARR/(ARPC×12))×CAC]/retention`; reserve 10-20% for experiments. Channel heuristics by client type (B2B SaaS = SEO+content+outbound; D2C = ASO+paid social; developer = technical content+DevRel; SMB = inbound, enterprise = outbound). 90-day roadmap in Unblock/Foundation/Velocity/Compound phases. Quality bar: explicit "Skip" decisions with rationale; missing data becomes "Open decision," never a guess. Three-phase workflow (INIT research/intake → REVIEW section-by-section approval → FINALIZE compile/verify) with resumable progress files.

**content-strategy** — Searchable-vs-shareable split (search demand-capture is the foundation; shareable creates demand). Buyer-journey keyword modifiers: awareness "what is/how to," consideration "best/vs/alternatives," decision "pricing/reviews/demo," implementation "templates/tutorial." Four-factor prioritization: Customer Impact 40% / Content-Market Fit 30% / Search Potential 20% / Resources 10%. 3-5 pillars (product-/audience-/search-/competitor-led) → topic-cluster maps.

**social (LinkedIn/X)** — Content pillar allocation (e.g., 30% insights / 25% BTS / 25% educational / 15% personal / 5% promo). Hook formulas: "I was wrong about [belief]"; "3 years ago I [past]. Today, [current]"; "How to [outcome] without [pain]"; "Unpopular opinion: [claim]". Repurposing math: 1 podcast → 3-5 clips + 1-2 LinkedIn posts + 1 thread + 1 carousel over 1-2 weeks. Cadence table (LinkedIn 3-5x/wk; X 3-10x/day). Mostly execution-cadence advice.

## 3. What to absorb vs skip for an automated website-analysis + positioning + per-ICP GTM pipeline

**Absorb — high value:**
1. **The hub-context architecture itself.** A single canonical positioning doc (their 12-section schema) that every downstream stage reads is exactly the right backbone for a pipeline. Their auto-draft-from-website-artifacts mode is literally your website-analysis step; the 12 sections are a strong extraction target schema.
2. **Four Forces (Push/Pull/Habit/Anxiety)** — compact, checkable switching-dynamics model for per-ICP positioning; rarely encoded elsewhere.
3. **Customer-research confidence labels (3+/2/1 sources, unprompted-weighted)** — directly automatable evidence-grading for scraped VOC; prevents single-review hallucinated ICPs. The persona schema (triggers/pains/outcomes/objections/alternatives/vocabulary/channels) is a good per-ICP record type.
4. **CRO 7-point priority-ordered audit + page-type rules + Quick Wins/High-Impact/Test triage** — converts website analysis into a scored, ordered, actionable report rather than a flat list.
5. **marketing-plan's constraint gating**: funding-stage budget unlocks, ARR-phase channel gates, channel-by-client-type heuristics, explicit "Skip with rationale," and "Open decision instead of guessing." These are the difference between a generic GTM doc and one a founder can execute; all are encodable as rules.
6. **Email sequence templates with counts/timing** and the copywriting headline/CTA formulas + 7-section page skeleton — deterministic scaffolds for per-ICP asset generation.
7. **Anti-personas + top-3 objections per ICP** — cheap to generate, high leverage for segmentation output.
8. **Content-strategy's buyer-journey keyword modifiers + weighted scoring** — automatable topic prioritization per ICP.

**Generic filler — skip:**
- **social**: cadence tables, daily-engagement routines, batching schedules — operational habits, not pipeline logic (the 4 hook formulas are worth keeping as a snippet).
- Copywriting's style rules (active voice, cut jargon) — any frontier model already does this; only the formulas/skeleton add value.
- Email body-length bands, "read aloud" tone advice, B2B-weekend timing — low-value conventional wisdom.
- Long tail of the repo (directory-submissions, sms, video, image, aso, schema, popups, co-marketing) — irrelevant to this pipeline.
- marketing-ideas' 140-tactic bank — useful only if cross-referenced through the AARRR/stage filter as marketing-plan does; raw, it's a listicle.

**Biggest single takeaway:** the repo's real innovation is structural, not content — gather positioning once into a schema'd doc, grade every claim by evidence strength, gate every recommendation by budget/stage constraints, and force explicit Skip/Open-decision calls. Those four mechanics map one-to-one onto an automated analysis→positioning→per-ICP-GTM pipeline.
